## Market Position

Nixelo sits at the intersection of three categories:

```
┌─────────────────────────────────────────────────────────────┐
│                         NIXELO                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   Project Management    +    Docs/Wiki    +    Meeting AI   │
│   (Linear, Jira)            (Notion)          (Read AI)     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## Categories

### Project Management
- **Linear** - Modern, fast, developer-focused
- **Jira** - Enterprise, feature-rich, complex
- **ClickUp** - All-in-one, feature-heavy
- **Asana** - Marketing/ops focused
- **Plane.so** - Open source Linear alternative

### Documentation
- **Notion** - Flexible docs + databases
- **Confluence** - Enterprise wiki (Atlassian)
- **Coda** - Docs + spreadsheets hybrid

### Meeting Intelligence
- **Read AI** - Meeting assistant + cross-platform search
- **Fireflies.ai** - Transcription + integrations
- **Otter.ai** - Real-time transcription
- **Gong.io** - Enterprise sales intelligence

### Infrastructure (Build vs Buy)
- **Recall.ai** - Meeting bot API ($0.50/hr)
- **Nylas** - Meeting + calendar APIs
- **Open source** - screenappai/meeting-bot, Vexa, Meetily

## Nixelo Differentiators

1. **Native integration** - Meetings → Issues → Docs all connected
2. **Real-time collaboration** - Not bolted on, built in
3. **Modern UX** - Linear-quality interface
4. **Meeting intelligence** - Only PM tool with built-in meeting AI
